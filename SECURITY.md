# Security Policy

## Reporting a Vulnerability

We take security issues seriously. If you discover a potential vulnerability in this project or its infrastructure, please report it using a **private GitHub issue** with the `security` label or via email to:

```
security@carcodez1.dev
```

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- Affected versions
- Any observed impact
- Suggested mitigation (optional)

We ask you **do not disclose the issue publicly** before maintainers have had a chance to respond.

## Acknowledgement

A maintainer will acknowledge your report within **72 hours** on business days and provide an expected timeline for resolution.

## Disclosure Policy

This project follows coordinated disclosure. Public disclosure should only occur after a patch or mitigation is available.

## Examples of Issues to Report

- Unauthorized access or privilege escalation
- Bypass of intended authorization boundaries
- Remote code execution vectors
- Sensitive data leakage
- Supply chain compromise (build or deployment pipeline)
